Mobile First Development

Web Design means designing for a myriad of devices and screen resolutions. Mobile First Development results in the best experience for every device.

Firstly, when it comes to designing a website, I am not sure if you should be designing the mobile version or desktop version initially. Perhaps, it's an overlapping process. For me, the design process overlaps the development process, and the mobile and desktop processes overlap. So, all aspects affect each other.

But, when it comes to the front-end development, you should develop the mobile friendly version first.

Here's Why

Progressive Enhancement. With the base CSS targeted for Mobile, you don't waste resources or bandwidth. Using Media Queries for larger screens, you can layer on CSS rules keeping the base as light as possible.

But, what about images and media?

There's a few solutions. This website uses simple jQuery checks for specific screen resolutions, replacing mobile-friendly images with higher resolution ones.

My fork of Jon Smith's Gridless Boilerplate allows for a Mobile First development process, just using Media Queries and very basic JavaScript statements. It's really easy to get started with this approach.

Designing websites means you need to consider many devices and screen sizes. Otherwise, you are failing as a web designer. Mobile First development is the best approach so you don't unnecessarily waste mobile users' bandwidth. Mobile First is just Progressive Enhancement emphasizing mobile-device compatibility.
